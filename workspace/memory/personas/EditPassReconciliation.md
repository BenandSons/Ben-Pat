# Edit Pass Reconciliation

## Purpose
This guide governs QA and enhancement passes where drafted copy must be reconciled against a source PDF, research handover, and an existing draft without losing structure or useful depth.

## Scope
This file is only for:
- edit-pass reconciliation
- preservation rules
- add-only enhancement logic
- FAQ merging
- tables and completeness checks

This file is not for:
- first-draft article ideation
- editorial-opinion strategy
- crypto intent mapping
- page-level SEO theory

## Inputs
Typical inputs:
- source PDF
- research handover
- draft copy

## Guardrails
- preserve section order unless there is a compelling reason not to
- preserve word count or increase it with useful additions
- prefer add-only enhancement rather than destructive rewriting
- do not hallucinate missing facts

## FAQ Policy
- merge FAQ material from source and research handover
- dedupe semantically similar questions
- expand the FAQ when obvious user questions are still missing
- place FAQs near the end, before sources, unless the format requires otherwise

## Shape Policy
- keep the structural shape of the strongest source document where sensible
- allow extra subsections when they improve clarity
- add tables when they materially improve sections such as:
  - costs and fees
  - timeframes and processing
  - document checklists

## Research Inclusion
- use all relevant facts from the research handover
- prefer official and stricter rules when sources conflict
- if a needed fact is missing, flag it explicitly instead of inventing it

## Completeness Checks
Depending on page type, confirm the draft includes:
- title
- meta summary
- key points
- eligibility and scope
- core rules or restrictions
- documents checklist
- costs and fees
- timeframes
- process steps
- common mistakes
- FAQs
- sources
- last reviewed or update marker where relevant

## Editing Checks
- maintain terminology consistency
- preserve factual precision
- remove AI tells and corporate filler
- tighten structure without flattening useful nuance
- ensure headings, tables, and anchors still line up cleanly

## Accessibility And Localization
- add alt text when images matter
- use table headers and captions where useful
- maintain locale-appropriate currency, dates, and units

## Output Shape
A good edit pass should report:
- summary of changes
- sections added
- tables added
- FAQs added or merged
- unused research facts
- issues fixed
- links checked
- open issues that still need verification

## What Adds Alpha
- reconciling more evidence into the draft without making it bloated
- preserving useful structure while improving accuracy
- exposing missing research instead of hiding it
- turning a draft into something publishable without losing its backbone
