# SwiftCargo International Localization Rules

## Purpose

Define how non-English SwiftCargo location pages should be created and edited so they can compete in their own language search markets without becoming literal translations of the English page.

## Core Principle

The English page is the structural source of truth.

That means:
- the locale page should generally preserve the same section order
- the locale page should preserve the same commercial purpose
- the locale page should preserve the same factual claims unless local adaptation is required
- the locale page should preserve the same conversion logic

But the English page is not the wording source of truth.

That means:
- translated sections do not need to mirror English sentence by sentence
- headings, metadata, CTAs, and body copy can be rewritten completely in the local language
- local keyword choices should be based on native-language competitiveness, not English phrasing mapped across mechanically
- examples, framing, and emphasis can shift when local search intent differs

## International SEO Rule

Every locale page should be judged as if it were competing directly against native-language pages already ranking in that market.

Do not ask:
- is this a faithful translation of the English page?

Ask:
- would this page compete credibly against strong native-language results for the target topic?
- does the wording sound like a native commercial page in that language?
- does the metadata match native-language search behavior?
- does the page answer the practical customs and relocation questions local users would actually search for?

## What Must Stay Aligned To English

These elements should usually stay aligned to the updated English page unless there is a strong locale reason not to:
- metadata intent
- hero intent
- primary CTA logic
- section order
- commercial objective
- customs and compliance truth conditions
- trust and risk framing
- conversion path

If the English page solves a structural weakness, the locale page should inherit that structural fix.

## What Can Change By Locale

These elements can and often should differ materially from English:
- title phrasing
- meta description phrasing
- hero wording
- keyword emphasis
- sentence structure
- paragraph rhythm
- examples and local framing
- subhead phrasing
- CTA language
- FAQ phrasing

A strong locale version may share the same structure while using substantially different copy.

## Translation Behavior Rules

Do not:
- translate sentence by sentence when that produces unnatural local-language SEO copy
- preserve English idioms that sound foreign in the target language
- copy English keyword stacking into another language
- keep weak English phrasing just for consistency
- over-localize facts that must remain operationally accurate

Do:
- write for native readability first
- rewrite headlines and sections when the local query pattern demands it
- keep customs, tax, and inspection claims defensible
- simplify awkward imported phrasing
- adapt tone so it feels like a serious local commercial page, not a translated export page

## Section-Level Editing Rule

For each locale page, review the page section by section against the updated English version.

For each section, ask:
1. does this section still serve the same job as the English section?
2. is the local-language version competitive in phrasing and intent?
3. should this section stay close to English, or should it be re-expressed more naturally for the locale?

The answer may differ section by section.

## Metadata Rule

Metadata must be localized for search competitiveness, not translated literally.

Requirements:
- include the destination and core service in natural local-language phrasing
- prefer search terms a native speaker would actually use
- avoid English syntax patterns in translated titles and descriptions
- keep the metadata commercially useful, not just accurate

## CTA Rule

CTA language should feel native and commercially sharp in the target language.

Allowed:
- rewrite CTA copy significantly from English
- shorten or sharpen CTA phrasing if the locale reads better that way

Not allowed:
- changing the conversion intent
- introducing a different offer than the English page implies

## Evidence Rule

When native-language search or performance data exists, use it.

When it does not exist:
- use the updated English page as the structural model
- make locale wording decisions by native-language readability and competitiveness
- label SEO conclusions as inferred where evidence is missing

## Minimum Standard For A Locale Pass

A locale page is not complete just because it has been translated.

A locale pass should leave the page:
- structurally aligned to the current English winner
- commercially coherent
- locally readable
- search-competitive in its own language
- factually consistent with customs and compliance realities

## Operational Rule For Future Requests

Requests like:
- update vietnam kr
- update australia fr
- update all remaining countries kr

should be treated as localization optimization work, not literal translation work.

The workflow is:
1. create today's dated locale file
2. compare it to the latest updated English working file
3. preserve English structure and conversion logic
4. rewrite sections in the target language for competitiveness where needed
5. validate JSON
6. save locale audit notes

## Locale Competitor And Keyword Rule

Every non-English page must have its own:
- competitor set
- keyword set
- SERP assumptions
- metadata phrasing
- page-specific keyword horizons for 3 months, 6 months, and 12 months

Do not reuse the English competitor set as if it were valid for the translated page.
Do not reuse the English keyword set as if it were the real search language of the target market.

English should supply:
- structure
- conversion logic
- factual constraints

The target language should supply:
- actual keyword targeting
- actual SERP competitors
- actual headline phrasing
- actual metadata language
- actual native-language intent fit

If native-language competitor research is missing, mark the locale SEO decisions as inferred rather than validated.

For competitor collection, aim for at least 10 pages when the target SERP supports it. Use a mixed evidence set rather than a single source class:
- official or regulatory sources when they materially shape the operational truth
- direct mover or removalist competitors
- adjacent ranking commercial or editorial pages that compete for the same local-language intent
