# Persona Usage Instructions

This document outlines the list of persona Markdown files and instructions for using them in writing or editing tasks for articles.

## Overview
The personas are based on influential writers and thinkers whose styles are incorporated into business writing to add depth, diversity, and impact. They should not be weighted equally. Strong article work follows persona tiers, with core writing priorities outranking later-added texture personas.

## Instructions for Writing/Editing Tasks
1. **Review Personas**: Before starting article work, review the Tier 1 set and any relevant Tier 2 set. Skim Tier 3 only if useful.
2. **Apply Priority Correctly**: Do not force equal weighting across all personas. Tier 1 outranks Tier 2, and Tier 2 outranks Tier 3.
3. **Use Personas to Strengthen, Not Decorate**: If a persona does not materially improve the article, leave it out and report it as not materially used.
4. **Reference Examples**: Use the example sections in persona files where helpful, but do not let mimicry override clarity or judgment.
5. **Track Usage**: After writing, note which personas or guides were materially used, lightly present, or not materially used.
6. **Iterate**: During editing, improve the weakest Tier 1 dimensions before adding more Tier 2 or Tier 3 texture.

## How To Use The Alpha Guides
1. Review the standard persona files for voice, framing, and intellectual style.
2. Review the Alpha Guides for durable optimization logic that should influence article decisions.
3. Treat the Alpha Guides as default inputs for VaaSBlock article work unless explicitly told otherwise.
4. Do not let Alpha Guides override thesis, evidence, or good judgment; they exist to sharpen strong content, not rescue weak content.

## Alpha Guide Roles
- `CryptoUserIntent.md`: choose realistic query targets, reader posture, and angle
- `EditorialDefensibility.md`: control claims, sourcing, attribution, and certainty
- `OnPageAlpha.md`: shape page structure, metadata, links, retrieval formatting, and implementation practicality
- `LongFormAuthorityArticles.md`: guide long-form editorial planning, host-site fit, competitor review, and article metadata
- `ProductPolicyPages.md`: guide operational/product-policy page structure, fast answers, and official-source formatting
- `EditPassReconciliation.md`: guide source-PDF and research-handover reconciliation passes
- `AIWritingTells.md`: run an editorial-quality cleanup for AI cadence, filler, and weak paragraphs
- `TransactionalEmail.md`: guide transactional quote-email flows, variable handling, and one-CTA operational messaging

This approach ensures articles are not only informative but also strategically weighted, editorially stronger, and less likely to collapse into persona-completion theater.
