# Page Memory

Use one Markdown file per page update so future refreshes start with the last known strategy instead of reconstructing it from scratch.

## What each page memory file should contain

- `page title`
- `local file`
- `live url`
- `site / brand`
- `publish history`
- `last major update`
- `core thesis`
- `primary target keywords`
- `secondary keywords`
- `latest GSC snapshot`
- `SERP / competitor notes`
- `key sources used`
- `internal links added or recommended`
- `what changed in the update`
- `current-to-improved scoring`
- `personas/guides materially used`
- `personas/guides reviewed but not materially used`
- `next update checks`

## Naming

Use a stable slug-style filename based on the live page topic, for example:

- `vaasblock-blockchain-industry-standards.md`
- `vaasblock-near-ai-blockchain.md`
- `swiftcargo-australia-biosecurity-evolution.md`

## Rule

Every substantial page rewrite or live-readiness pass should create or update its matching page memory file in this folder.
